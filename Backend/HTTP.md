# HTTP

- HTML을 서로 주고받을 수 있도록 만들어진 프로토콜

---
## HTTP 통신 방식
![](https://blog.kakaocdn.net/dn/2uDtD/btqEepoz93p/NdnyQBPeYLS18pxzO32xck/img.jpg)

- 클라이언트가 서버로 request를 보내고, 서버가 response를 클라이언트에게 보낸다.
- Stateless : 요청과 응답하는 정보가 저장되지 않아 이전에 요청과 응답 과정을 기억하지 않는다. (독립적)

---
## 쿠키와 세션
- 쿠키(cookie) : 웹 브라우저가 웹 사이트에서 보내온 정보를 저장할 수 있도록 하는 작은 파일,
클라이언트가 필요한 정보를 포함해서 보낼려면 클라이언트가 정보를 저장 할 수 있는 메커니즘이 필요
- 세션(session) : 쿠키와 같은 메커니즘이지만 차이점은 쿠키는 클라이언트에서 데이터를 저장하지만 세션은 서버에서 데이터를 저장한다.

## HTTP 요청 구조
- Start Line
    - HTTP 메소드 : 해당 HTTP 요청이 의도하는 액션을 정의하는 부분<br>
    GET 요청은 서버로부터 데이터를 받고자 할때, POST 요청은 서버에 새로운 데이터를 저장할 때 사용된다.
    - Request target : 해당 HTTP 요청이 전성되는 목표 주소
    - HTTP version : 해당 요청의 HTTP 버전
- Header 
    - Host : 요청이 전송되는 target의 호스트의 URL주소를 알려 주는 헤더
    - User-Agent : 요청을 보내는 클라이언트의 대한 정보
    - Accept : 해당 요청이 받을 수 있는 응답(response) body 데이터 타입을 알려 주는 헤더
    - Connection : 해당 요청이 끝난 후에 클라이언트와 서버가 계속해서 네트워크 연결을 유지할 것인지, 끊을 것인지에 대해 알려 주는 헤더
    - Content-Type : HTTP 요청이 보내는 메시지 body의 타입을 알려 주는 헤더
    - Content-Length : HTTP 요청이 보내는 메시지 body의 총 사이즈를 알려 주는 헤더
- Body 
    - HTTP 요청이 전송하는 데이터를 담고 있느 부분
    - 전송할 데이터가 없다면 body 부분은 비어 있다.

## HTTP 응답 구조
- Status Line 
    - HTTP version : HTTP의 버전
    - Status Code : HTTP 응답 상태를 미리 지정되어 있는 숫자로 된 코드를 나타낸다.<br>
    정상적으로 처리가 되면 200이 나타난다.
    - Status Text : HTTP 응답 상태를 간략하게 글로 설명해 주는 부분<br>
    정상적으로 처리가 되면 "OK"를 나타낸다.
- Header 
    - 요청의 헤더와 동일
- Body
    - 요청 메시지의 body와 동일

## 자주 사용되는 HTTP 메소드
- GET
    - 데이터를 서버로부터 요청할 때 주로 사용하는 메소드
    - 데이터의 생성이나 수정, 삭제 등의 변경 사항이 없이 단순히 데이터를 받아 오는 요청 시
- POST
    - GET 메소드와 함께 사용되는 메소드
    - 데이터를 생성하거나 수정, 삭제 요청을 할 때 사용
- OPTIONS
    - 특정 엔드포인트에서 허용하는 메소드들이 무엇이 있는지 알고자 하는 요청에서 사용

---
## 자주 사용되는 HTTP Stauts Code
- **200 OK** : 요청이 문제없이 성공적으로 처리 되었을 때
- **301 Moved Permanetly** : HTTP 요청을 보낸 엔드포인트의 URL 주소가 바뀌었음을 나태내는 코드
- **400 Bad Request** : HTTP 요청이 잘못된 요청일 때, 요청에 포함된 값들이 잘못된 값들이 보내졌을 때 나타난다.
- **401 Unauthorized** : HTTP 요청을 처리하기 위해서는 해당 요청을 보내는 주체(사용자 혹은 클라이언트)의 신분(credential) 확인이 요구되나 확인할 수 없었을 때 나타난다.<br>
주로 해당 HTTP 요청을 보내는 사용자가 로그인이 필요한 경우 401 응답을 보낸다.
- **403 Forbidden** : 요청을 보내는 주체가 해당 요청에 대한 권한이 없을 때
- **404 Not Found** : 요청을 보내고자 하는 URI가 존재하지 않을 때
- **500 Internal Server Error** : 내부 서버 오류가 발생했다는 것을 알려 주는 코드, 요청을 받은 서버에서 해당 요청을 처리하는 과정에서 서버 오류가 나서 해당 요청을 처리할 수 없을 때
